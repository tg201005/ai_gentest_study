

> 파일명: 개인정보파일의 대량발송 자동화에서 배운 QA 설계의 중요성

## Situation (상황)

- **문제 상황:** 수십개의 조합 별로 수신자, 수십명에게 각각 문서 PDF를 첨부하여 메일을 발송해야 했음. 현재 문서는 하나의 통합된 파일로 존재하는 상황임. 메일 발송시 제목, 본문, 파일, 참조, 숨은 참조 등을 통제해야 하는 상황. 
- **사용 도구:** 자동화 도구, Claude code
- **배경:** 2025년도 정기 발송용 문서를 조합원별로 개별 첨부파일과 함께 발송하는 반복 업무. 수동 발송 시 수 시간 소요

## Task (과제)

- **목표:** CSV 기반 수신자 리스트 + 디렉토리 내 개별 첨부파일 매칭 + 공통 첨부파일 포함하여 대량 메일 자동 발송
- 목표: 어떻게 코드를 일일히 확인하지 않고 해당 메일 프로세스 전반에 걸친 무결성을 증명할 수 있을까?


## Action (행동)

### 시행착오

- 개별 첨부파일을 이름으로 매칭하는 로직에서, 파일명과 CSV 내 이름이 정확히 일치하지 않는 케이스 발생
- 자동화 도구 자동으로 발송까지 진행하려는 것 확인. 
- 테스트 발송 없이 바로 보내면 사고 위험이 높음을 인지

## 해결 방법
- 발송 전 후를 기준으로, 전체 프로세스를 발송 계획 목록 작성 부분과 발송 부분으로 분리함.  발송계획목록 생성과 발송 함수 맞춤화까지는 claude code에 위임하고, 계획서의 매칭을 필자가 직접 확인하고, 발송 버튼을 직접 누르는 방식으로 진행. 
- 테스트용 데이터를 함께 생성하도록 워크플로우 설계
- 테스트 발송 대상자를 지정하여 사전 검증

## Result (결과)

- +)
	- 문서 개인 LP들에게 개별 첨부파일 포함 메일 자동 발송 완료 → **성공**
	- 대량 메일 발송 워크플로우 자동화 템플릿 양식 구성 완료
- -)
	- 10만명의 메일 자동발송화라면 인간 검수가 가능하지 않다. 
	- 그때마다 만들어지는 메일 발송 함수의 무결성도 어떻게 보장할 수 있는가?

## Learning (배움)

- **결정적 활용팁:** 워크플로우 자동화에 있어서 어떤 기능 및 동작이 해당 워크플로우 신뢰성에 가장 중요한지를 인식하고 분할 정복하는 것이 필요하다. 
- **배운 것:** 
	- +) 워크플로우 자동화에 있어서 어떤 기능 및 동작이 해당 워크플로우 신뢰성에 가장 중요한지를 인식하고 분할 정복하는 것이 필요하다. 
	- - ) 어떻게하면 각 단계에서의 무결성을 보장할 수 있을까? 
		- -> 만약 가능한 99.9%의 케이스를 만들어 테스트 할 수 있다면?
		- -> 만약 모든 모듈이 테스트 할 수 있게 설계돼있다면?
