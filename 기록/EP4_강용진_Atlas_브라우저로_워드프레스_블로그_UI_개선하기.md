# Atlas 브라우저로 워드프레스 블로그 UI 개선하기

> **작성자:** 강용진
> **날짜:** 2026-03-14
> **에피소드:** EP4 (4주차)

---

## Situation (상황)

- **문제 상황:** 원포인트 블로그의 UI를 개선해야 하는 상황이었다. 그런데 이 블로그는 워드프레스 기반이고, 나는 워드프레스 개발을 제대로 진행해본 적이 없었다. 일반적인 프로젝트라면 IDE에 연결해서 에이전틱 AI(Cursor, Claude Code 등)로 코드를 수정하면 되지만, 워드프레스는 웹 페이지 내에서 직접 수정해야 하는 구조라 기존 방식을 적용하기 어려웠다.
- **사용 도구:** OpenAI Atlas (AI 브라우저)
- **배경:** 워드프레스 경험이 없는 상태에서, 어디서 무엇을 어떻게 수정해야 하는지조차 파악이 안 됐다. IDE 연동이라는 선택지가 있었을 수도 있지만, 당시에는 그 생각까지 미치지 못했다.

## Task (과제)

- **목표:** AI 브라우저인 Atlas를 활용하여 워드프레스 기반 블로그의 UI를 마케터가 제공한 레이아웃에 맞게 개선한다.

## Action (행동)

### 시행착오

- Atlas를 떠올리기 전에는 어떻게 접근해야 할지 막막했다. 워드프레스의 관리자 패널 구조, 테마 커스터마이징 방식, CSS 수정 위치 등을 전혀 모르는 상태였다.
- 처음에는 ChatGPT에 텍스트로 물어보면서 수정을 시도했다. 하지만 "어디 메뉴에 들어가서 어디를 눌러라"는 식의 안내를 받아도, 실제 화면과 매칭이 안 되거나 버전 차이로 메뉴가 다른 경우가 있어 시행착오가 있었다.

### 해결 방법

**1. Atlas 브라우저 도입**

웹 페이지를 AI가 직접 보면서 안내해주는 Atlas 브라우저를 사용하기로 했다. 텍스트로 설명을 주고받는 것이 아니라, AI가 실제 화면을 보고 어디를 클릭해야 하는지, 어떤 코드를 어디에 넣어야 하는지를 알려주는 방식이다.

**2. 피그마 레이아웃을 레퍼런스로 제공**

마케터분이 피그마로 만들어준 블로그 레이아웃을 Atlas에 보여주면서, "이 디자인대로 수정하고 싶다"고 요청했다. Atlas가 피그마의 레이아웃과 현재 워드프레스 블로그 화면을 비교하며, 어디서 무엇을 어떻게 수정해야 하는지를 단계별로 안내해줬다.

**3. AI 안내에 따라 수정 적용**

Atlas가 알려주는 대로 워드프레스 관리자 패널에서 테마 설정, CSS 커스텀 코드, 위젯 배치 등을 수정했다. 워드프레스를 처음 다루는 상황이었지만, AI가 실제 화면을 보면서 "여기를 클릭하고, 이 코드를 넣어라"고 구체적으로 안내해줬기 때문에 큰 어려움 없이 진행할 수 있었다.

## Result (결과)

- 워드프레스 경험이 전혀 없는 상태에서, 마케터가 제공한 피그마 레이아웃에 맞게 블로그 UI를 개선했다.
- 텍스트 기반 ChatGPT 대화로는 막막했던 작업을, Atlas 브라우저를 통해 화면을 직접 보면서 해결할 수 있었다.
- IDE 없이 웹 브라우저만으로 UI 수정 작업을 완료했다.

## Learning (배움)

- **결정적 활용팁:** IDE로 연결하기 어려운 웹 기반 플랫폼(워드프레스, 카페24 등)의 코드 수정에는 AI 브라우저(Atlas)가 효과적이다. 특히 해당 플랫폼 경험이 없을 때, AI가 실제 화면을 보면서 안내해주므로 텍스트 기반 대화보다 훨씬 정확하고 빠르다.
- **배운 것:**
    - 텍스트로 "어디를 수정해라"라는 안내를 받는 것과, AI가 실제 화면을 보면서 "여기를 클릭해라"라고 안내하는 것은 정확도와 속도에서 큰 차이가 있다.
    - 에이전틱 AI를 활용할 수 없는 환경이라고 판단하더라도, AI 브라우저처럼 다른 형태의 AI 도구가 대안이 될 수 있다. 도구 선택의 폭을 넓게 갖는 것이 중요하다.
    - 워드프레스처럼 웹 관리자 패널에서 수정하는 플랫폼은, IDE 연동보다 AI 브라우저가 더 자연스러운 작업 흐름을 제공할 수 있다.
