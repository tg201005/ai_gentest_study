# Pencil MCP로 디자인 도메인 없이 대시보드 UI/UX 개선하기

> **작성자:** 강용진
> **날짜:** 2026-03-07
> **에피소드:** EP3 (3주차)

---

## Situation (상황)

- **문제 상황:** 원포인트 대시보드를 사용하는 고객사 유저들이 UI 이용 시 헷갈려하는 부분이 있었다. 이로 인해 서비스 전환율에 악영향을 끼치고 있다는 가설을 정의했고, UI/UX 개선이 필요하다고 판단했다.
- **사용 도구:** Pencil MCP (Cursor 익스텐션)
- **배경:** UI/UX를 개선해야 하는 상황인데, 피그마를 다루지 못하고 디자인 전문성도 없는 1인 개발자 상황이었다. 디자인 외주나 디자이너 협업 없이 빠르게 개선안을 만들어야 했다.

## Task (과제)

- **목표:** 고객사 유저의 혼란을 줄이는 방향으로 대시보드 UI를 개선하고, 이를 바로 프론트엔드 코드로 연결하여 빠르게 배포한다. 디자인 도구 없이, AI와의 대화만으로 원하는 디자인을 뽑아낸다.

## Action (행동)

### 시행착오

- 처음에는 텍스트로만 UI 개선 방향을 설명하며 코드를 바로 작성하려 했다. 하지만 머릿속에 있는 UI를 텍스트로 정확히 전달하기가 어렵고, AI가 만들어준 결과물을 실제로 렌더링해서 확인하기 전까지는 맞는지 알 수 없었다.
- 결과물이 마음에 들지 않으면 코드를 수정하고 다시 확인하는 루프가 길어져, UI 개선에만 지나치게 많은 시간이 소요될 것이 우려되었다.

### 해결 방법

**1. Pencil MCP 도입**

Cursor 익스텐션으로 Pencil MCP를 설치했다. 설치 방법은 Cursor 익스텐션 마켓에서 검색 후 설치하면 MCP 등록까지 자동으로 완료되어 매우 간편했다.

**2. 레퍼런스와 지침을 함께 제공하며 디자인 대화**

- 참고하고 싶은 UI 레퍼런스(스크린샷 또는 설명)를 Pencil MCP에 제공했다.
- 개선하고자 하는 화면의 문제점과 원하는 방향을 지침으로 함께 전달했다.
- Pencil MCP가 `.pen` 파일로 디자인 시안을 생성하면, 결과물을 보고 피드백을 주며 원하는 방향으로 계속 다듬었다.

**3. 디자인 확정 후 바로 코드 작성**

- Pencil MCP와의 대화로 시안이 확정되면, 해당 디자인을 그대로 프론트엔드 코드로 옮겼다.
- 코드 작성 시 이미 시각적으로 확인된 디자인을 기반으로 하므로, 핑퐁 없이 한 번에 완성도 높은 코드를 만들 수 있었다.

## Result (결과)

- Pencil MCP와 지속적으로 대화하며 원하는 UI가 나올 때까지 디벨롭할 수 있었다. 코드를 한 줄도 짜기 전에 시각적 결과물을 먼저 확인하고 수정할 수 있어, 전체 개선 사이클이 크게 단축되었다.
- 확정된 디자인을 기반으로 프론트엔드 코드를 바로 작성해 빠르게 배포했다.
- 피그마, 디자인 전문 지식, 디자이너 협업 없이 UI/UX 개선을 완수했다.

## Learning (배움)

- **결정적 활용팁:** 디자인 도메인이 없는 1인 개발자라면, 코드를 짜기 전에 Pencil MCP로 시각적 시안을 먼저 잡아라. 레퍼런스와 문제점을 구체적으로 설명할수록 원하는 결과를 빠르게 얻을 수 있다.
- **배운 것:**
    - 디자인 능력이나 피그마 숙련도가 없어도, 레퍼런스와 명확한 지침만 있으면 AI와의 대화로 충분히 원하는 UI를 만들어낼 수 있다.
    - Pencil MCP는 무료이고, Cursor 익스텐션 설치만으로 바로 사용할 수 있어 진입 장벽이 매우 낮다. 비용 부담 없이 시도해볼 수 있다는 점이 1인 개발 환경에서 특히 큰 장점이다.
    - UI/UX 개선은 "코드 수정 → 확인" 루프가 아니라 "디자인 확정 → 코드 작성" 순서로 접근하면 전체 사이클을 훨씬 줄일 수 있다.
