# RCP Shader Graph의 커스텀 속성 지원 부재로 인한 바리센트릭 와이어프레임 구현 실패

## Situation (상황)

- **문제 상황:** Blender에서 생성한 바리센트릭(Barycentric) 데이터를 활용하여 Reality Composer Pro(RCP)에서 정교한 와이어프레임(Wireframe) 머티리얼을 구현하고자 함.
- **사용 도구:** ChatGPT (GPT-5.2)
- **배경:** visionOS 앱 개발 중 메쉬의 폴리곤 에지를 강조하는 시각 효과가 필요하여, 일반적인 셰이더 기법인 바리센트릭 좌표계를 적용하려 함.

## Task (과제)

- **목표:** Blender에서 구운 Vertex Color(바리센트릭 데이터)를 RCP Shader Graph에서 읽어와 에지 라인을 계산하고 출력하는 머티리얼 노드 구성.

## Action (행동)

### 시행착오
- **시도:** `Geometry Property` 노드를 사용하여 Blender에서 내보낸 Vertex Color 레이어 이름을 입력하고 데이터를 가져오려 함.
- **결과:** RCP Shader Graph 내에서 해당 데이터를 정상적으로 읽어오지 못함.
- **원인:** RCP의 Shader Graph는 MaterialX 표준을 따르지만, 현재 버전에서는 **커스텀 버텍스 속성(Primvars)이나 버텍스 컬러를 직접 참조할 수 있는 입력 노드가 인터페이스상에 구현되어 있지 않음.** (표준에는 정의되어 있으나 Apple의 현재 구현체에서 누락됨)

### 해결 방법
- **수정:** 확인(Verify) 후 방법(How-to)
- **방법:**
    1. 확인: "이 노드/기능이 RCP에 실제로 있니?" (Is it there?)
       - AI가 "당연히 되겠지"라고 가정하지 못하게 '지원 여부'를 먼저 묻기
       - 질문 예시:
         > Reality Composer Pro의 Shader Graph에서 Blender의 Vertex Color나 Custom Primvars(속성)를 직접 읽어올 수 있는 노드가 현재 존재하나요? 만약 없다면, 바리센트릭 데이터를 활용한 와이어프레임 구현이 가능한지, 아니면 다른 우회 방법을 써야 하는지 알려주세요.
       - 효과: AI가 노드 리스트를 검색하게 만들어 전용 노드의 부재를 먼저 인지
    2. 제약: "이 방식의 한계가 뭐니?" (What are the limits?)
       - '안 되는 부분'을 먼저 확인
       - 질문 예시:
         > visionOS용 RCP Shader Graph는 일반적인 MaterialX와 달리 메쉬 데이터 접근에 제약이 많다고 들었습니다. Blender에서 만든 바리센트릭 데이터를 활용해 와이어프레임을 만들고 싶은데, 현재 RCP에서 이 방식이 기술적으로 가능한가요? 안 된다면 RCP 환경에서 와이어프레임을 구현하는 가장 표준적인 방법은 무엇인가요?
       - 효과: AI가 '일반적인 이론'이 아니라 'RCP의 특수성'에 집중
    3. 방법: "그럼 현재 가능한 최선의 구현법은 뭐니?" (What's the best way now?)
       - 특정 기술(바리센트릭)에 매몰되지 않고 '목적(와이어프레임)'에 집중
       - 질문 예시:
         > RCP Shader Graph에서 와이어프레임 효과를 구현하고 싶습니다. 바리센트릭 좌표를 쓰는 방식이 RCP 노드 구조상 가능한지 확인해 주시고, 만약 버텍스 데이터를 읽는 노드가 없다면 UV 조작이나 텍스처 베이킹 등 RCP에서 실제로 작동하는 대안을 단계별로 알려주세요.
       - 효과: 바리센트릭 방식이 불가능할 경우, AI가 즉시 "그건 안 되지만 이 방법은 됩니다"라고 올바른 방향을 제시

## Result (결과)

- **최종 결과:** Shader Graph 단독으로 바리센트릭 데이터를 읽는 것은 불가능함을 확인
- **성공 여부:** 부분 성공 (초기 목표인 '바리센트릭 데이터 직접 활용'은 툴의 제약으로 실패했으나, 시스템적 불가능 인지).

## Learning (배움)

- **결정적 활용팁:** **"확인(Verify) 후 방법(How-to)"** 전략을 통해 AI의 논리적 비약을 방지하고 툴의 실제 제약 사항을 먼저 파악해야 한다.
- **배운 것:** 
    1. **질문 순서의 중요성:** AI는 특정 표준(MaterialX)을 근거로 "당연히 지원될 것"이라 추론하므로, **"이 노드가 실제로 존재하는가?"**를 먼저 묻는 확인(Verify) 단계가 필수적이다.
    2. **플랫폼 특수성 인지:** Reality Composer Pro와 같은 신생 플랫폼은 타 엔진(Unity, Unreal)의 상식이 통하지 않는 제약이 많으므로, '구현 방법(How-to)'을 묻기 전 '지원 범위'를 먼저 확정 짓는 질문 습관이 오류를 줄이는 핵심이다.
    3. **대안 사고:** 특정 기술적 통로(바리센트릭)가 막혔을 때, 목적(와이어프레임 시각화)을 달성할 수 있는 다른 데이터 채널(UV, Texture)을 빠르게 탐색해야 한다.
