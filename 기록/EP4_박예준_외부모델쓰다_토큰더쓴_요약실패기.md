## Situation (상황)

* 문제 상황:
  긴 수업 자료를 요약해야 했는데, GPT 토큰 사용량을 줄이고 싶어서 외부 summarization 모델을 먼저 사용하려고 했다.

* 사용 도구:
  Hugging Face summarization 모델, GPT, 로컬 실행 환경

* 배경:
  수업 자료 PDF와 노트가 많아서 한 번에 GPT에 넣으면 토큰이 많이 들 것 같아서, 먼저 외부 모델로 요약한 뒤 GPT에 넣으면 더 효율적일 것이라고 생각했다.

---

## Task (과제)

* 목표:
  여러 개의 수업 자료를 효율적으로 요약하면서 토큰 사용량을 최소화하는 방법 찾기

---

## Action (행동)

### 시행착오

* Hugging Face의 summarization 모델을 사용해서 먼저 문서를 요약하려고 했다.
* 긴 문서를 그대로 넣었더니 입력 길이 제한 때문에 잘려서 요약이 이상하게 나왔다.
* 문서를 나눠서 여러 번 요약했는데, 결과가 서로 연결되지 않았다.
* 결국 요약된 결과를 다시 GPT에 넣어 정리해야 했다.
* 중간 요약 + 최종 요약을 하다 보니 오히려 호출 횟수가 늘어났다.

왜 잘 안 됐는지

* 외부 모델은 문맥을 길게 유지하는 데 한계가 있었다.
* 요약 결과를 다시 정리해야 해서 추가 토큰이 들었다.
* 한국어 요약 품질이 일정하지 않았다.

---

### 해결 방법

* 문서를 외부 모델로 먼저 요약하는 대신,
  GPT에 직접 넣되 작은 단위로 나눠서 요약했다.
* 주차별로 요약 → 전체 통합 요약 → 시험용 요약 순서로 진행했다.

사용한 방식

* 프롬프트:
  "이 자료를 시험 대비용으로 핵심 개념 중심으로 요약해줘"
  "중복 내용은 합치고 중요한 개념만 남겨줘"
  "전체 흐름이 보이도록 정리해줘"

* 방법:

  1. 자료를 여러 개로 나눔
  2. 각각 요약
  3. 요약본을 다시 넣어서 통합

---

## Result (결과)

* 최종 결과는 GPT만 사용했을 때가 더 깔끔했다.
* 외부 모델을 거쳤을 때보다 토큰 사용량도 크게 차이 나지 않았다.
* 오히려 시행착오 때문에 총 사용량은 더 늘어났다.

결과: 부분 성공 (방법은 찾았지만 처음 전략이 비효율적이었음)

---

## Learning (배움)

결정적 활용팁:
토큰을 아끼려고 외부 모델을 쓰면, 중간 정리 과정 때문에 오히려 더 많이 쓸 수 있다.

배운 것:

* 긴 문서는 한 번에 요약하려 하지 말고 나눠서 처리하는 게 좋다.
* 범용 LLM + 단계별 요약이 대부분의 경우 더 안정적이다.
* 외부 summarization 모델은 자동화에는 좋지만 학습용 정리에는 부족할 수 있다.
* 최적화하려다가 구조를 복잡하게 만들면 비용이 더 늘 수 있다.
