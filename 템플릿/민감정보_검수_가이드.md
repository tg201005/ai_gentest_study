## 2. 스크린샷/이미지 체크리스트

스크린샷을 첨부하기 전 아래 항목이 노출되지 않는지 확인하세요:

- [ ] 회사 로고/브랜드명
- [ ] 내부 시스템 UI (메뉴, 네비게이션 포함)
- [ ] URL 바에 내부 도메인
- [ ] 실제 데이터 (이름, 금액, 날짜 등)
- [ ] 터미널/CLI에 파일 경로, 환경변수
- [ ] 이메일 주소, 수신자 정보

> 위 항목 중 하나라도 포함되면 **마스킹 처리하거나 첨부하지 마세요.**

## 3. 작성 원칙

### 핵심: "시행착오의 구조"를 공유하되, "어디서 했는지"는 숨긴다

좋은 예:
> 소속 회사에서 레거시 스프레드시트 데이터를 관리할 내부 페이지가 필요했다.
> CC에게 한 줄로 지시했더니 기존 데이터 구조와 맞지 않았다.

나쁜 예:
> A벤처스에서 투자 심사 데이터 300건을 Supabase로 마이그레이션하는 어드민을 만들었다.
> LP 40명에게 연말정산 투자확인서를 Antigravity로 발송했다.

### 자기 점검 질문

1. **이 문장만 보고 내 회사/업종을 특정할 수 있는가?** → 수정
2. **회사 동료가 보면 "이거 우리 시스템 아닌가?" 할 수 있는가?** → 수정
3. **경쟁사가 보면 사업 전략을 유추할 수 있는가?** → 수정
4. **제3자의 개인정보가 포함되어 있는가?** → 삭제

## 4. 제출 전 최종 체크

```
grep -iE "회사명|서비스명|내부도구명|LP|펀드|투자|심사|조합원|확인서|연말정산" 파일명.md
```

위 명령어의 회사명/서비스명 부분을 본인의 실제 회사명으로 바꿔서 실행하세요.
**결과가 0건이어야 제출 가능합니다.**
