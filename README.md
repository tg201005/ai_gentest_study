# AI 스터디 - 에이전틱 AI 활용 시행착오 탐색 및 공유

에이전틱 AI(Claude Code, OpenClaw 등 CLI 기반 AI 에이전트)를 실무와 개인 프로젝트에 활용하면서 겪는 시행착오를 구조적으로 기록하고, 서로 공유하며, 그 결과물을 온라인에 공개하는 스터디입니다.

| 폴더         | 용도                                                        |
| ------------ | ----------------------------------------------------------- |
| **Episode/** | 매주 스터디 미팅 후 정리된 **최종 회의록**이 쌓이는 공간    |
| **기록/**    | 멤버 각자의 **시행착오 기록**(STARL)이 공유되는 공간        |
| **회의록/**  | 스크립트, Gemini 회의록 등 **가공 전 원문**이 보관되는 공간 |
| **템플릿/**  | 기록 및 회의록 **작성용 템플릿**                            |

---

## 목적

1. **에이전트 AI 활용에 관한 시행착오를 탐색하고 시도하며, 그 과정을 공유한다.**
2. **그 결과물을 온라인상에 누구나 볼 수 있도록 배포한다.**

> "어떻게 하면 에이전틱 AI를 의도대로 통제하고, 누가 써도 일관된 품질의 결과물을 효율적으로 얻을 수 있는가?"

## STARL 템플릿

시행착오를 일관되고 구조적으로 공유하기 위한 형식:

| 섹션          | 내용                                    |
| ------------- | --------------------------------------- |
| **Situation** | 어떤 문제 상황이었나? 어떤 도구를 썼나? |
| **Task**      | 해결해야 할 과제/목표                   |
| **Action**    | 시행착오 → 해결 방법                    |
| **Result**    | 결과                                    |
| **Learning**  | 결정적 활용팁, 배운 것                  |

상세 템플릿: [`템플릿/STARL_템플릿.md`](템플릿/STARL_템플릿.md)

## 스터디 규칙

### 스터디 전

- 스터디 당일 자정(00:00)까지 기록 MD 파일 제출 필수
- 다른 멤버들의 기록을 최소 1회 이상 읽고 오기

### 스터디 중 (80분 / 인당 20분)

1. 청중이 먼저 **10분간 질문**
2. 발표자가 질문에 답하며 **추가 맥락 설명** (10분)

### 스터디 후

- 영상 편집 → 유튜브 업로드
- 회의록 정리 → Episode 폴더에 보관

## 운영 원칙

1. **일단 하는게 중요하다** — 완벽한 설계보다 빠른 실행과 반복 개선
2. **거창하지 않아도 된다** — 작은 발견이라도 공유하면 가치가 있다
3. **시행착오가 곧 학습이다** — 실패한 시도도 기록하고 공유한다
4. **구조는 유연하게** — 디렉토리 구조, 템플릿 등은 매주 의견을 받아 개선한다
