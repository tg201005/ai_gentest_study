## 합의 사항

### 1. 스터디 목적

> **에이전트 AI 활용에 관한 시행착오를 탐색하고 시도하며, 그 과정을 공유한다.**
> **그 결과물을 온라인상에 누구나 볼 수 있도록 배포한다.**

- 학습에 대한 기대: 시행착오를 통한 AI 능력 범위 확인 및 상호 학습
- 기여와 브랜딩에 대한 기대: 텍스트 + 영상 결과물을 통한 공유 및 배포

### 2. 산출물 정의

| 구분 | 산출물 |
|------|--------|
| **개인별** | STARL 템플릿 기반 MD 파일 (주차별 1개) |
| **팀별** | 스터디 영상 녹화 → 유튜브 업로드 |
| **회의록** | Google Gemini 회의록 → 레포지토리 보관 |

### 3. 공유 방식: GitHub 오픈 레포지토리 (마크다운 기반)

**노션 대신 GitHub + MD를 선택한 이유:**

| 장점 | 단점 |
|------|------|
| AI가 로컬 파일에 직접 CRUD 가능 (AI 친화적) | 실시간 편집 및 공유가 어려움 |
| SEO에 친화적 | |
| 확장성이 좋음 (인원 추가, 주제 확장 등) | |

- 레포지토리 생성: 담당자 배정
- 디렉토리 구조: 초기 임의 세팅 후, 주마다 의견을 받아 수정

### 4. STARL 템플릿

시행착오를 구조적으로 공유하기 위한 템플릿:

```
## Situation
- 어떤 문제 상황이었나요?
- 어떤 도구를 사용했나요?

## Task
- 해결해야 할 과제나 목표는 무엇이었나요?

## Action
- 시행착오: 처음에는 어떻게 했나요?
- 해결 방법: 어떤 도구와 환경을 사용해서 어떻게 수정했나요?

## Result
- 결과는 어땠나요?

## Learning
- 결정적 활용팁:
- 배운 것:
```

### 5. 스터디 진행 규칙

**스터디 전:**
- 스터디 당일 00:00까지 MD 파일 제출해야 참석 가능
- 다른 분들의 시행착오를 적어도 한 번씩은 확인하고 오기

**스터디 중 (80분, 인당 20분):**
1. 청중이 먼저 **10분간 질문**을 던진다
2. 발표자가 질문에 답하며 **추가 맥락을 설명**한다 (10분)
- 사전에 MD를 읽어왔다는 전제 하에 진행

**스터디 후:**
- 영상 편집 → 유튜브 업로드
- 회의록 정리 → 레포지토리 디렉토리에 보관

## 공통 키워드

회의 중 참석자 전원에게서 반복적으로 나온 핵심 관심사:

- **일관성 (Consistency):** 누가 써도 일관된 품질의 결과물을 얻는 방법
- **통제 (Control):** AI를 의도대로 정확하게 제어하는 소통 양식
- **비용 효율 (Cost Efficiency):** 적은 비용으로 0→80을 만드는 전략
- **시행착오 공유 (Shared Learning):** 개인 경험을 구조적으로 공유하여 학습 가속화
